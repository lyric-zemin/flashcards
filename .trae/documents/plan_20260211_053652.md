# 重构计划：提取header为通用组件并实现layout包裹容器

## 1. 创建通用Header组件
- **文件路径**: `src/components/Header.tsx`
- **功能**: 支持不同页面的header需求
- **Props设计**:
  - `title`: 页面标题
  - `showBackButton`: 是否显示返回按钮
  - `backButtonText`: 返回按钮文本
  - `onBackClick`: 返回按钮点击事件
  - `showAuthButtons`: 是否显示认证相关按钮（仅首页需要）
  - `children`: 额外的子元素（可选）
- **样式要求**:
  - 使用`position: sticky`定位，保持在页面顶部
  - 适配手机端显示，避免文字错位和按钮变形
  - 使用响应式设计，确保在不同屏幕尺寸下正常显示

## 2. 创建Layout布局组件
- **文件路径**: `src/components/Layout.tsx`
- **功能**: 统一页面布局结构
- **结构**:
  - 主要内容区域
  - Footer组件
  - **注意**: 不包含Header组件，Header将在每个页面中单独使用

## 3. 重构App.tsx
- **修改内容**: 使用Layout组件包裹所有路由
- **实现方式**: 为每个路由添加Layout组件包装

## 4. 更新页面组件
- **移除**每个页面中的重复footer代码
- **保留**主要内容
- **使用**新的Header组件替代原有的header代码：
  - HomePage.tsx
  - FlashcardListPage.tsx
  - FlashcardDetailPage.tsx
  - ProfilePage.tsx
  - GameModePage.tsx
  - LoginPage.tsx
  - RegisterPage.tsx

## 5. 验证重构结果
- 确保所有页面的header使用sticky定位，滚动时保持在顶部
- 确保在手机端显示正确，无文字错位和按钮变形
- 确保返回按钮功能正常
- 确保认证按钮在首页正确显示
- 确保整体布局结构一致

## 技术要点
- 使用React functional components
- 利用props传递动态内容和事件处理
- 保持组件的可重用性和灵活性
- 遵循现有的CSS类名规范（使用unocss预设类名）
- 添加响应式设计，确保在不同屏幕尺寸下正常显示
- 使用sticky定位替代固定定位，提供更好的用户体验
